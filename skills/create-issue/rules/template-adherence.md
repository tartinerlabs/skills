**Rule**: When issue templates exist in the repository, follow the exact template structure. When no templates exist, use a clear custom format.

### With Template

1. Identify the most appropriate template (bug report, feature request, etc.)
2. Parse template structure and required sections
3. Fill in all required fields with relevant information from user context
4. Preserve template formatting and section headers

### Without Template

Use a clear, structured format:

```markdown
## Problem
[Clear problem statement or feature description]

## Context
[Steps to reproduce (bugs) or additional details (features)]
```

Keep it concise and actionable.
